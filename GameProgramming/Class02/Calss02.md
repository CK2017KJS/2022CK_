~~고급?~~

- Game데이터의 자료형

    일단 정수형만을 관리하는 경우는 거의 없다. 필연적으로 실수를 사용해야 할 상황이 온다.
    사용 가능한 자원이 좆망하면 고정 소수점을 사용하는것도 고려해 볼 것

- 조건문

    조건문을 쓰는거야 상관이 없는데, 코딩 스타일을 준수해서 작성하자.
    교수님 추천으로는 IDE가 자동적으로 권장해주는 경우를 1순위에 두자.

- 중첩 조건문

    있으면 편한 기능이고 쓰면 편하지만 코드가 꼬여버리면
    디버그시에도 어디인지 알기 쉽지 않아서 좆망 해버리니까 가독성을 위주로 두자.

- 반복문
    
    반복문 할때 반복을 관리하는걸 i나 j로 퉁치는데
    플밍 공부는 몰라도 게임 개발상에선 그렇게 하지 말 것. 
    가독성을 좀 중요하게.. if문이나 for문의 다음 문장이 한줄인 경우에는
    대괄호가 필요 없지만 그런 암시적인 표현도 자제할 것

- 이중 반복문

    반복문의 시간 복잡도를 고려할 것. 또한 게임 프로그래밍은 이미
    게임 자체가"반복문"이므로, 다중 반복문의 사용은 자제할 것,
    최악의 상황은 비효율적인 다중 반복문을 매 틱레이트마다 실행하는 것

- foreach

    C#의 고유 문법..
    이라곤 해도 파이썬의 for A in IndexA 와 완전히 같다.
    for문이 조금 더 빠르다. 가독성과 퍼포먼스중에 잘 고민해볼 것

- 클래스

    그러니까 뉴비들이 정적 메서드에 대한 이해를 못하지
